Comment by Dmitrii Prisacari [ 12/Apr/17 ]

Приветствую, yybitb. В рамках этого задания нужно всю известную документацию включить в состав репозитория. Так будет удобно менять доки одновременно с кодом при выполнении заданий и в случае удаления файлов с гугл-диска, доки будут сохранены. Предлагаю сделать это следующим образом.

1. Нужно немного изменить расположение файлов в репозитории. В конечном виде репозиторий выглядит так:

  • src - директория со всем кодом (это все, что сейчас есть в репозитории)
  • docs - директория с документацией
  • README.md - файл с оглавлением документации

2. Переписать документацию в файлы формата markdown. Это лучше, чем варианты *.docx и *.rtf, потому что содержимое markdown можно будет отслеживать через git.

При написании доков нужно использовать шаблон по написанию технического задания. Шаблон рассчитан на проекты любой сложности. Серым цветом выделена информация с подсказками, что писать в каком разделе. В случае, если какой-то раздел шаблона неприменим для проекта, его можно удалить или оставить пустым, но в него дописать причину, почему раздел пуст. Например: "данные не выявлены".

В доки можно сразу включить всю информацию по вопросам, которые я задавал.
