 Description   

Необходимо переработать логику отправки email. Будем делать рассылки по заданным настройкам, то есть менеджер будет указывать одну и более настроек (фильтров) рассылки кому отправлять. Настройки будут такие :

  • Все домены: "Да" , "Нет" или "Список" то
    Text area показывается : сюда вбивается список доменов: каждый с новой строки, либо через запятую.
  • Email: можно указать через запятую емейлы или с новой строки .
  • Дата создания домена с: DD-MM-YYYY по: DD-MM-YYYY
  • Дата последней отправки email с: DD-MM-YYYY по: DD-MM-YYYY
  • Язык вебмастера: русский, английский и т.д. ( см. тикет MBC-12 Done ) (мультиселект )
  • По : Email или WhoIs Email ( мультиселект )
  • По категориям доменов: MP3, Кинотеатр, Торренты и т д (мультиселект )
  • По формату рекламы : popunder, skin, banner (мультиселект )
  • По доменам пользователя , опционально только для админа (мультиселект )
  • По Geo : указываем к примеру RU>50 , отправка идет только таким доменам
  • По трафику (поле Visits) указываем <>= трафик: к примеру >100000 , отправка идет только тем у кого столько трафика

После того как сформирован фильтр отправки писем , вносится текст письма, указывается тема письма, нужно его выводить в табличку в которой будет фиксироваться вся информация про эту рассылку.

Пример:

Название рассылки Дата создания Дата оконачания рассылки Отправлено Stop/start
(тема письма) DD-MM-YYYY DD-MM-YYYY 10/1000 start/stop кнопка

Тему письма делаем кликабельной, и по клику можно просмотреть детали по фильтрам которые использовались для этой рассылки, процесс отправки, с указанием каким доменам/email было отправлено письмо , дату отправки.
Рассылка писем начинается только после нажатия Start в этой таблице
